## Phase 7.1 v1.0 — 콘솔 자동완성(prompt hint) 시나리오 설계서 (복붙용)

```md
# Phase 7.1 v1.0 — Console Prompt Hints (Autocomplete) 설계서

목표
- 사용자가 "순서/개념/버튼"을 배우지 않아도, 한 문장으로 봇을 움직이게 만든다.
- 내부 용어(collect/analyze/job 등)는 노출하지 않는다.
- 힌트는 "명령어 목록"이 아니라 "사람에게 말하는 예시 문장"이어야 한다.

핵심 UX 원칙
1) 한 줄로 끝난다: 시간 + 목적 + 결과가 한 문장에 들어간다.
2) 결과 중심: "리포트/요약/브리핑/알림" 같은 출력 단어를 앞세운다.
3) 다음 행동이 보인다: 사용자가 지금 무엇을 해야 하는지 힌트가 알려준다.
4) 자동화는 선택: "매일/매주" 같은 반복은 사용자가 원할 때만.
5) 안전장치 포함: 외부 게시/발송/댓글은 항상 "승인"을 전제로.

---

## 1) 힌트 표시 위치(필수)
A. 콘솔 입력창 placeholder
- 예: "사람에게 말하듯 한 문장으로 시켜보세요. 예) 내일 아침 9시에 시장 리포트 만들어줘"

B. 입력창 포커스 시 "힌트 드롭다운"
- 최근 사용 힌트 3개 + 추천 힌트 7개 (총 10개)
- 키보드 ↑↓ 선택, Enter로 입력창에 삽입

C. 빈 화면(메시지 0개) 온보딩 카드
- "한 문장으로 시작하기" 섹션 + 예시 3개 + '삽입' 버튼

D. 실행 결과 후 "다음으로 할 일" 카드
- 수집 완료 / 초안 생성 / 승인 대기 상태에 따라 다른 힌트 제공

---

## 2) 힌트 카테고리(필수 6종)
각 카테고리는 4~8개 예문을 갖는다.

(1) 시작(First Run)
- 아직 아무것도 모르는 사용자가 첫 성공을 맛보게 하는 문장

(2) 설정/자동화(Schedule)
- 매일/주간/시간 변경 같은 반복 설정

(3) 즉시 실행(One-time Run)
- 오늘만, 지금만, 테스트로 한 번 같은 실행

(4) 범위/필터(What to focus)
- 미국 주식/크립토/환율/금리/지정학 등 범위 지정

(5) 출력 스타일(Output style)
- 짧게/자세히/핵심만/뉴스 앵커 톤 등 (말투는 옵션이지만 구조는 남겨둔다)

(6) 안전/승인(Safety)
- 승인 후 발송/외부 공유 금지/논쟁 회피 등

---

## 3) 상태 기반 힌트(가장 중요)
콘솔은 항상 현재 상태를 보고 추천 힌트를 바꿔야 한다.

상태 S0: 봇이 선택되지 않음(또는 active bot 없음)
- 힌트: "내 봇 목록 보여줘", "투자 봇으로 전환해줘"

상태 S1: 봇은 있는데 소스가 없음(또는 0개)
- 힌트: "기본 소스 넣고 시작해줘", "이 URL 소스를 추가해줘: ..."

상태 S2: 소스는 있는데 최근 수집이 없음
- 힌트: "지금 자료 모아서 요약해줘", "테스트로 오늘 리포트 만들어줘"

상태 S3: 초안이 생성됨(승인 대기)
- 힌트: "초안 보여줘", "수정해서 다시 만들어줘", "승인할게"

상태 S4: 스케줄은 켜졌는데 결과가 안 옴(사용자 혼란 구간)
- 힌트: "왜 리포트가 안왔는지 점검해줘", "다음 실행 시간 알려줘", "지금 한 번 실행해줘"

---

## 4) 기본 힌트 문장(Investing Bot 템플릿)
아래 문장은 그대로 UI에서 사용한다.

### (1) 시작(First Run)
- "오늘 시장 자료 모아서 요약 리포트 만들어줘"
- "투자 봇을 매일 아침 9시에 브리핑하게 설정해줘"
- "처음이니까 테스트로 한 번만 리포트 만들어줘"
- "내 설정이 잘 됐는지 점검하고 필요한 걸 알려줘"

### (2) 설정/자동화(Schedule)
- "매일 아침 9시에 리포트 받게 해줘"
- "리포트 시간을 아침 8시로 바꿔줘"
- "주말은 빼고 평일만 리포트 해줘"
- "하루 1번만, 과하게 돌리지 말고 해줘"

### (3) 즉시 실행(One-time Run)
- "지금 바로 오늘 리포트 한 번 만들어줘"
- "최근 24시간 이슈만 빠르게 요약해줘"
- "이번 주만 테스트로 매일 돌려줘"

### (4) 범위/필터(What to focus)
- "미국 주식 위주로만 정리해줘"
- "크립토 시장도 같이 포함해줘"
- "금리/환율/원자재 영향까지 한 줄로 정리해줘"
- "국제 정세가 시장에 미치는 영향 위주로 봐줘"

### (5) 출력 스타일(Output style)
- "뉴스 앵커처럼 깔끔하게 핵심만"
- "오늘은 좀 자세히, 근거 중심으로"
- "한 줄 요약 + 핵심 5개만"

### (6) 안전/승인(Safety)
- "내 승인 없이는 어디에도 게시하거나 발송하지 말아줘"
- "확실한 출처만 쓰고, 추측이면 추측이라고 표시해줘"
- "논쟁적인 표현은 피하고 중립적으로 정리해줘"

---

## 5) 자동완성 구현 요구사항(기술)
- 프론트: hint preset 목록을 JSON으로 관리(카테고리/상태/토픽별)
- 백엔드: 현재 스레드/봇 상태를 반환하는 /api/console/context endpoint 제공
  - activeBotId, sourceCount, lastCollectedAt, pendingDraftCount, scheduleNextRunAt 등
- 프론트: context를 5~10초마다 polling 또는 메시지 이벤트 후 갱신
- 힌트 선택 로직:
  1) 상태 기반(top 4)
  2) 토픽 기반(Investing 등) 기본(top 4)
  3) 최근 사용(최대 3)
- 사용자가 힌트를 선택하면 입력창에 삽입만 하고 자동 실행하지 않는다.
  - 실행은 사용자가 Enter로 보내는 순간만

---

## 6) 성공 기준(테스트 시나리오)
- 신규 사용자: 힌트 1개 클릭 → Enter → 60초 안에 첫 결과(요약/초안/다음 행동)가 나온다.
- 소스 0개: "기본 소스 넣고 시작" 힌트로 소스가 추가된다.
- 리포트 안 옴: "왜 안왔는지 점검" 힌트로 원인이 설명된다(시간/스케줄/키/소스 등).
```

## 덤: 콘솔 힌트에 꼭 들어가야 할 “헷갈림 해결” 3문장

사용자들이 가장 자주 막히는 구간을 찌르는 문장입니다.

* **“내가 뭘 해야 하는지 모르겠어. 다음 단계 알려줘”**
* **“왜 리포트가 안 왔는지 점검해줘”**
* **“지금 한 번만 테스트로 실행해줘”**

이 3개는 어떤 토픽 봇이든 “생명줄”이에요.
