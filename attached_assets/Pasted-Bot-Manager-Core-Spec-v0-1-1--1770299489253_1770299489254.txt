Bot Manager – Core Spec v0.1

(범용 리서치/커뮤니티/리포트 봇 매니저)

1. 이 프로젝트의 목표

이 시스템은 한 사람이 쓰는 개인 봇이 아니라,

여러 사용자가
각자 원하는 목적(리서치, 리포트, 커뮤니티, 모니터링 등)에 맞춰
“자기만의 봇”을 조립해서 쓰는 범용 봇 매니저를 만드는 것이 목표다.

핵심 철학은 다음과 같다.

모든 사용자가 같은 데이터, 같은 리포트를 받지 않게 한다

사용자가 직접

소스를 고르고

중요도를 정하고

스케줄을 정하고

출력 형태를 조절한다

봇을 조정하고 최적화하는 재미가 있어야 한다

말투/스타일은 나중에 옵션으로 추가한다

지금은 기능과 구조가 먼저다

2. 시스템의 기본 구조 (Pipeline)

모든 봇은 아래 파이프라인을 공유한다.

Collect

RSS, 피드, 소스에서 콘텐츠 수집

Analyze

수집된 아이템을 분석

중요도, 주제, 중복 여부, 위험도 등을 점수화/태깅

Output

봇의 목적에 따라 결과가 달라진다

리포트형: Report 생성

커뮤니티형: Draft(댓글 초안) 생성

모니터링형: 알림/변경 요약 생성

즉:

어떤 봇이든
“Collect → Analyze → (Draft 또는 Report 또는 Alert)” 구조를 가진다.

3. Topic(토픽) 개념

모든 데이터는 Topic으로 분리된다.

예:

investing (시장/투자)

ai_art (AI 아트 커뮤니티)

tech

policy

custom …

Topic의 역할:

소스 분리

분석 결과 분리

리포트/초안 생성 시 섞이지 않게 필터링

“AI Art 리포트에 투자 뉴스가 들어가는 사고”를 구조적으로 방지

모든 Source와 Item은 반드시 topic을 가진다.

4. Preset(프리셋) 개념

Preset은 “봇의 목적 템플릿”이다.

예시 프리셋:

Daily Market Brief (리포트형)

Competitor Watch (모니터링형)

Community Engagement Helper (커뮤니티 참여형)

Preset은 다음을 정의한다:

어떤 Output을 만드는가 (Report / Draft / Alert)

어떤 기본 정책을 쓰는가

어떤 설정 항목들이 필요한가

5. Variant(변형) 개념

Variant는 Preset의 세부 용도다.

예:

Preset: Community Engagement Helper

Variant: ai_art

Variant: tech

Variant: startup

Variant: faith

즉:

“AI Art 봇”은 독립 프리셋이 아니라

Community Engagement Helper의 Variant 중 하나다

6. User Profile(사용자 프로필) 개념

사용자는 같은 Preset이라도 여러 개의 프로필을 만들 수 있다.

Profile에 들어가는 것:

선택한 Sources 목록

중요도 가중치 (예: macro 40, equities 30, crypto 30)

Output 옵션 (길이, 섹션 포함 여부 등)

스케줄 (시간대, 트리거)

Topic

결과:

같은 “Daily Market Brief”라도

사람마다 다른 소스

다른 중요도

다른 시간

다른 길이
→ 다른 리포트를 받게 된다

7. Sources(소스) 설계 원칙

시스템은 “추천 소스 번들”을 제공한다

예: “신뢰도 우선”, “속보 우선”, “매크로 중심”

하지만:

사용자는 체크박스로 끄고 켤 수 있어야 한다

사용자는 자기 RSS를 추가할 수 있어야 한다

Sources는 반드시 topic을 가진다

즉:

시스템은 “정답”을 강요하지 않고

“좋은 출발점”만 제공한다

8. Daily Market Brief (리포트형) – 역할 정의

이 프리셋은:

Draft(댓글) 생성 ❌

Alert ❌

오직:

Analyze된 아이템 중

topic = investing 인 것만 모아서

하루 1회 Report 생성

특징:

Report는 사용자 Profile 설정에 따라

포함 소스

중요도

길이

포함 섹션

생성 시간
이 달라진다

9. 스케줄 시스템

각 Profile마다 스케줄을 가진다

예:

매일 22:00 KST

미국장 개장 1시간 전

주 1회 월요일 아침

서버는:

Profile 스케줄을 기준으로

해당 Profile의 설정을 읽어서

그 Profile 전용 Output을 생성한다
