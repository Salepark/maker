즉, Phase 4 v1.0 “기능”은 끝났습니다.

2) 지금 당장 해야 하는 “마감 작업” 6개 (헷갈림 제거용)

이 6개만 하면, 앞으로는 “어디까지가 Phase 4였지?”가 안 생깁니다.

A. UX/제품 일관성 2개

용어 통일

UI에서 “Profiles”라는 단어가 남아있으면, 사용자 혼란이 큽니다.

권장: Templates / Presets / Gallery 중 하나로 완전히 통일

프리셋 vs 봇 vs 토픽

위저드에서 토픽 선택이 들어갔다면:

프리셋이 “multi-topic”인지 “single-topic”인지 UI에서 명확히

예: “이 템플릿은 여러 토픽 중 선택 가능합니다.”

B. 데이터/운영 안정성 2개

시드 중복 방지 확인

seeds가 여러 번 실행되어도 presets가 중복으로 늘어나지 않게(UNIQUE key)

이미 key unique면 OK

source url 중복 처리

findSourceByUrl 추가했으니:

이미 존재하는 소스면 재사용할지/새로 만들지 정책을 명확히

보통은 “봇-소스 연결은 새로 만들고, 소스 자체는 재사용”이 깔끔합니다.

C. 리포트/LLM 경계 2개

Report LLM 선택 우선순위 문서화

지금은 profile → topic → bot 매칭해서 bot LLM 사용하도록 브릿지 했죠.

우선순위가 문서로 박혀 있어야 나중에 안 꼬입니다.

예: bot-specific LLM > system default > (없으면 에러)

System Status 위젯에서 사용자 행동 안내

“시스템 키 없음”이면 단순 표기 말고,

“BYO LLM을 Providers에서 추가하세요” 같은 행동 유도 한 줄

3) “획일화 방지”가 Phase 4의 핵심인데… 지금 구현이 살짝 걱정되는 1가지

당신 구현 요약에 따르면:

추천 소스가 “모두 pre-selected” 상태

이건 사용자가 빨리 시작하기엔 좋지만, 모두가 거의 같은 리포트를 받는 방향으로 갑니다(당신이 싫어하던 지점).
그래서 **Phase 4.0.1(초소형 패치)**로 아래 한 줄만 바꾸는 걸 강력 추천합니다:

추천 소스 기본 선택을 0~2개만 체크(나머지는 제안)

또는 “Recommended(기본 2개)” / “Optional(나머지)”로 그룹 나누기

이 한 줄이 “플랫폼의 재미”를 살립니다.
