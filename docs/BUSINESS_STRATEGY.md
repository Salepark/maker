# Maker 사업 전략 및 철학 문서

> **Maker Desktop App은 신뢰 기반 플랫폼으로 키운다**
>
> **우리는 AI를 믿지 않습니다. 우리는 사용자의 통제권을 믿습니다.**

---

# I. 선언문

> Maker는 AI를 위한 OS가 아니라, **사용자를 위한 AI 통제 시스템이다.**

> AI는 많다. **운영체제는 아직 없다.**

> 우리는 봇을 판매하지 않습니다. **우리는 운영체제를 만듭니다.**

---

# II. 핵심 철학

> Maker는 기술 데모가 아니라, **철학이 있는 OS**
>
> 기능이 많은 OS가 아니다. 속도가 빠른 OS도 아니다.
> **원칙이 흔들리지 않는 OS다.**

AI 시대에 가장 무너진 것: **통제권(Control)**. Maker는 이걸 되찾는 운영체제다.

### 1. 통제는 항상 사용자에게 있다
- AI는 허락받고 움직인다.
- 자동화는 승인받고 실행된다.

### 2. 결과는 반드시 보장된다
- Running...으로 방치하지 않는다.
- **Fast-first. Always.**

### 3. 데이터는 사용자의 자산이다
- 웹 SaaS는 데이터를 흡수한다.
- Maker는 데이터를 사용자 옆에 둔다.

### 4. 자동화는 인간을 대체하지 않는다
- 자동화는 인간의 의사결정을 보조한다.
- Maker는 AI를 "상사"로 만들지 않는다.
- **AI는 도구다.**

### 5. 숨기지 않는다
- 무엇을 수집했고, 무엇을 분석했고, 무엇을 보냈는지 항상 기록한다.
- **Audit 로그가 있는 이유다.**

---

# III. 왜 OS인가?

| 운영체제의 특징 | Maker가 가진 것 |
|---------------|----------------|
| 권한 관리 | Permission Engine (11개 권한) |
| 자원 관리 | Local DB (자원 통제) |
| 프로세스 관리 | Pipeline Engine |
| 실행 기록 | Job Runs |
| 안전한 실행 환경 | Fast-first 리포트 (실행 보장) |

> **이미 OS의 구조다.**

### Maker OS의 4대 원칙

| 원칙 | 의미 |
|------|------|
| **Local First** | 데이터는 당신의 컴퓨터에 |
| **Permission by Design** | AI는 항상 허락받고 움직인다 |
| **Fast-First Execution** | 결과는 즉시 제공된다 |
| **Human-in-Control** | 자동이지만, 통제는 인간에게 |

---

# IV. 경쟁 포지셔닝

### 시장 내 위치

| 제품 | 포지션 | 특징 |
|------|--------|------|
| Make / n8n | 자동화 빌더 | 연결 중심 |
| Zapier | SaaS 연결기 | 단순 연결 |
| 몰트봇 | 자율 에이전트 | LLM 중심, 서버 의존, 비용 누적 |
| **Maker** | **개인 자동화 OS** | Local-first, 통제 중심 |

### Maker의 비교우위

| 무기 | 설명 |
|------|------|
| **Local-first** | 몰트봇: "우리가 대신 해준다" / Maker: "당신이 소유한다" |
| **Permission Engine** | 11개 권한, 승인 시스템, 감사 로그, Egress 제어 |
| **Fast-first Report** | 즉시 Fast Report, Background upgrade, Never empty |
| **Controlled Autonomy** | 몰트봇은 자율성 강하지만 통제 약함. Maker는 반대 |

### 이 철학이 곧 USP

기능과 속도는 따라 만들 수 있다. 하지만 **"통제 중심 AI OS"** 포지션은 쉽게 따라 못 한다.

대부분 SaaS는 데이터를 수집해야 돈이 되고, 중앙 통제가 비즈니스 모델이기 때문이다.
**Maker는 반대다.**

# VI. 핵심 메시지

> **당신의 데이터는 당신의 것이다. 당신의 자동화도 당신이 통제한다.**

### 브랜드 문장 세트

| 용도 | 문장 |
|------|------|
| **Hero** | AI 시대, 당신만의 자동화 운영체제가 필요합니다. |
| **Subline** | Maker는 AI를 연결하고, 권한을 통제하고, 리포트를 보장하는 개인 자동화 OS입니다. |
| **Strong** | 우리는 봇을 판매하지 않습니다. 우리는 운영체제를 만듭니다. |

### Elevator Pitch (30초)

Maker는 당신의 데이터, 파일, 일정, 리포트를 AI와 연결하는 **개인 자동화 운영체제**입니다. SaaS에 종속되지 않고, 당신이 권한을 통제하며, AI를 실제로 "일하게" 만듭니다.

### 투자자 피치

> 우리는 또 하나의 AI 툴을 만드는 것이 아닙니다.
> 우리는 AI 시대의 **"개인 인프라 레이어"**를 만듭니다.
> SaaS가 아닌, 사용자 중심의 자동화 플랫폼입니다.

# VII. 장기 비전

### Jarvis 진화 경로

```
Maker OS
  → 권한 기반 자동화
  → 로컬 데이터 연결
  → 지속적 학습 메모리
  → 개인 행동 패턴 이해
  → Context-aware Assistant
```

- 몰트봇은 "능력"을 키운다.
- **Maker는 "통제된 능력"을 키운다.**
- 이 차이가 장기 승부를 가른다.

> 무통제 AI는 Jarvis가 아니다. 그건 그냥 자동화 엔진이다.

### 3년 로드맵

| 시기 | 목표 |
|------|------|
| **1년차** | Desktop Core 확산, 개발자 커뮤니티 확보, 플러그인 생태계 시작 |
| **2년차** | Pro 확장, 개인 자비스 고도화, Power User 유료 전환 |
| **3년차** | 기업용 보안 자동화 OS, 규제 친화적 AI 플랫폼, "Local-first AI Infra" 브랜드 확립 |

# VIII. 현실 평가

### 리스크

- 초기에 돈이 안 된다
- 성장 속도가 느릴 수 있다
- "와!" 하는 데모가 덜 화려하다
- 투자자에게 설명이 어렵다
- 마케팅이 어렵다

### 하지만

- **한 번 신뢰를 얻으면 이탈이 거의 없다**
- 신뢰 기반 플랫폼으로 확장 가능
- 장기 체류 사용자 확보
- 프리미엄 모델 가능
- **성공하면 대체 불가능한 카테고리를 차지한다**

---

# IX. 제품 방향 원칙

1. 로컬 권한 모델을 끝까지 밀어붙이기
2. 데이터 Egress 통제를 강력하게 유지
3. 절대 "몰래 자동 실행" 금지
4. 빠른 리포트 신뢰도 100% 확보

> Maker는 AI를 더 강하게 만드는 게 아니라,
> **AI를 안전하게 다루는 플랫폼**이 된다.

---

# X. 최종 정의

> Maker는 "AI 자동화 도구"가 아니다.
> **Maker는 AI 시대의 개인 자동화 운영체제다.**
>
> Maker는 AI에게 일을 시키는 도구가 아니라,
> **핵심 철학의 구현체다.**
