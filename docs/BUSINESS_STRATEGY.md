# Maker 사업 전략 문서

> Maker Desktop App은 신뢰 기반 플랫폼으로 키운다

---

## 1. 사업 구조

이건 "기술 결정"이 아니라 **사업 구조를 결정하는 문제**

### 1-1. 데스크톱 앱 오픈소스 공개 vs 구독형 배포

#### A. 데스크톱 앱을 오픈소스로 공개할 경우

**장점**
- **신뢰 확보**: "로컬에서 모든 데이터를 처리한다", "백도어 없다" — AI 시대에 이건 강력한 메시지
- **커뮤니티 확장**: 플러그인, 소스 커넥터, 자동화 레시피 생태계 가능. 개발자 참여로 확산
- **빠른 기능 확장**: 외부 기여로 커넥터, 에이전트 기능 증가
- **보안 브랜드화 가능**: "We don't own your data."

**단점**
- **유료화가 어려워짐**: 데스크톱이 무료면 웹 구독 유인이 약해질 수 있음
- **포크 리스크**: 누군가 Maker 기반 SaaS를 만들어버릴 수 있음 (라이선스로 어느 정도 방어 가능)
- **지원 비용 증가**: 오픈소스는 질문이 많아짐, 유지관리 리소스 소모

#### B. 데스크톱을 구독형/유료 배포

**장점**
- **명확한 수익 모델**: Pro Desktop, Enterprise Desktop, 연간 라이선스
- **기능 제어 가능**: 고급 권한 / Local FS / Full LLM Egress 등 Pro 전용
- **지원 범위 통제**: 무료 유저 대응 최소화

**단점**
- **신뢰 장벽**: "로컬인데 닫혀 있다?" — AI 자동화 OS에서 이건 불리함
- **확산 속도 느림**: 초기 사용자 기반 확대가 더딜 수 있음
- **커뮤니티 형성 어려움**

## 2. 전략적 선택지 (현실적 제안)

가장 현실적인 구조:

### Core 오픈소스 + Desktop Pro 유료

**오픈소스 범위 (Core):**
- 자동화 엔진
- 권한 시스템
- 기본 리포트

**유료 범위 (Pro):**
- 고급 권한
- 장기 메모리
- 자비스 모드
- 자동 업그레이드
- 클라우드 동기화

> 이게 가장 균형 잡힌 모델

---

## 3. Maker Desktop vs 몰트봇 같은 에이전트

### 몰트봇 계열의 특징
- LLM 중심, 웹 기반, API 중심
- 에이전트 자율성 강조, 서버 의존
- 비용 누적형 구조

**장점:** 강력한 실행 자동화, 빠른 기능 확장, 이미 투자/마케팅 강함

**약점:** 데이터가 서버로 감, 장기 안정성 불확실, 비용 구조 불투명, "플랫폼 종속"

### Maker Desktop의 비교우위

#### 1. Local-first 아키텍처
- 몰트봇: "우리가 대신 해준다"
- Maker: "당신이 소유한다"
- **이건 철학 차이**

#### 2. Permission Engine
- 11개 권한, 승인 시스템, 감사 로그, Egress 제어
- 몰트봇은 이런 세밀한 통제 모델이 없음
- **기업용 확장 시 매우 강력**

#### 3. Fast-first Report 구조
- 즉시 Fast Report, Background upgrade, Never empty
- 많은 에이전트 툴이 여기서 무너짐

#### 4. Automation OS 포지션
- 몰트봇 = AI Agent Tool
- **Maker Desktop = Personal Automation OS**
- 이 차이는 큼

## 4. 가장 중요한 질문

> Maker는 AI에게 일을 시키는 도구입니까,
> 아니면 **개인 자동화 운영체제**입니까?

후자라면: 데스크톱은 단순 제품이 아니라 **핵심 철학의 구현체**

---

## 5. 전략적 제안

초기 단계 전략:

- **Desktop Core는 오픈소스** — 신뢰 + 확산
- **Pro 기능은 유료** — 장기 수익
- **웹은 별도 분리** — 협업, 조직 기능, 클라우드 동기화
